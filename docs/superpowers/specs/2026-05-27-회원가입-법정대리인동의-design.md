# 회원가입 — 법정대리인 동의 플로우 프로토타입 설계

- 날짜: 2026-05-27
- 대상: CopyKiller School 회원가입 (`https://testschool.copykiller.com/signup`)
- 목적: 만 14세 미만 어린이회원 가입 시 **법정대리인(보호자) 휴대폰 본인인증 + 동의** 절차를 추가한 클릭형 프로토타입

## 배경

기존 회원가입은 약관 동의 → 정보 입력 흐름이다. 만 14세 미만 어린이회원의 경우, 법령상 법정대리인의 동의가 필요하므로 유형 선택 직후 보호자 본인인증 단계를 끼워 넣는다. 보호자가 직접 본인인증을 완료하면 그 결과로 보호자 이름·연락처가 채워지고 편집 불가가 된다.

## 산출물

- 단일 자체완결형 HTML 파일: `회원가입_법정대리인동의_프로토타입.html` (레포 루트)
- inline `<style>` + 바닐라 JS. `foundation.css`의 컬러 토큰을 일반 CSS 변수로 이식해 톤앤매너 일치.
- 외부 의존성: Pretendard 폰트 CDN 1개만.

## 톤앤매너 (foundation.css 기반)

- 폰트: Pretendard
- 프라이머리 블루: `#00c2ff` (primary-500), hover `#0097d4` (primary-600)
- 텍스트: `#3e4448` (slate), 보조 `#7d7d80`
- 배경: `#f4f5f6`, 카드 흰색, 라운드 12px
- 레드/경고(필수 안내): `#ff006b` 계열, 핑크 배경 `#fff1f4`
- 비활성 버튼: `#cdd1d5` 회색

## 화면 레이아웃

- 연블루~그레이 배경 위 중앙 카드 (max-width 480px), 모바일 우선.
- 카드 상단: `CopyKiller SCHOOL` 로고 + 4단계 스텝 인디케이터.
- 단계 전환은 섹션 show/hide (JS 상태머신), 인증은 모달 오버레이.

## 플로우 (어린이회원 기준 4단계)

### Step 1 — 회원가입 유형 선택
- 제목 "회원가입", 안내문 "가입 유형에 따라 필요한 인증 및 정보 절차가 다릅니다."
- 선택 카드 2개: `일반회원(만 14세 이상)`, `어린이회원(만 14세 미만)` — 선택 시 블루 테두리 + 연블루 배경.
- `다음`: 선택 전 비활성.
- 분기: 어린이회원 → Step 2. 일반회원 → Step 2 건너뛰고 Step 3(약관)로 (간단 분기).

### Step 2 — 법정대리인 동의 및 인증 (어린이회원 전용)
- 핑크 안내박스: "만 14세 미만 회원은 법정대리인 동의가 필요합니다. 보호자 인증 후 다음 단계로 진행할 수 있습니다."
- `보호자 이름*` 입력 (인증 전 비활성/placeholder).
- `보호자 전화번호*` 입력 + `인증하기` 버튼.
- `인증하기` → 휴대폰 본인인증 모달.
- 인증 성공 시: 보호자 이름·전화번호 **자동 기입 + 편집 불가(readonly) + 인증완료 뱃지**.
- `(필수) 법정대리인으로서 아동의 개인정보 수집 및 이용에 동의합니다.` 체크박스.
- `다음`: **인증완료 AND 동의 체크** 시에만 활성. `취소` 버튼(Step 1로).

### Step 3 — 약관 동의
- 제목 "약관 동의", 안내문 "선택 항목은 동의하지 않아도 이용할 수 있으나 일부 기능이 제한될 수 있습니다."
- `전체 동의` 마스터 체크박스.
- `(필수) 서비스 이용 약관` / `(필수) 개인정보 수집 및 이용 동의` / `(선택) 마케팅 정보 수신 동의` — 각 우측 `>`(상세 토글).
- `다음`: 두 필수 항목 체크 시 활성. `취소`.

### Step 4 — 어린이회원 정보 입력
- 제목 "회원정보 입력".
- 아이디 (+ `중복확인` 버튼 모킹) / 비밀번호 / 비밀번호 확인.
- 이름 / 생년월일.
- 학교 / 학년.
- `가입완료` → 완료 화면("회원가입이 완료되었습니다").

## 휴대폰 본인인증 모달 (모킹)

- 헤더 "휴대폰 본인인증".
- 통신사 select (SKT/KT/LG U+/알뜰폰).
- 보호자 이름 / 생년월일 입력.
- 휴대폰 번호 입력 + `인증번호 발송`.
- 인증번호 6자리 입력 (힌트: `123456`, 타이머 표시).
- `인증 완료` → 모달 닫힘, 입력한 이름·휴대폰번호를 Step 2 보호자 필드로 write-back.

## 상태/버튼 활성화 규칙 요약

| 단계 | 다음 버튼 활성 조건 |
|------|--------------------|
| Step 1 | 유형 1개 선택 |
| Step 2 | 보호자 인증완료 AND 법정대리인 동의 체크 |
| Step 3 | 필수 약관 2개 체크 |
| Step 4 | (프로토타입) 가입완료 클릭 시 완료 화면 |

## 비범위 (YAGNI)

- 실제 PASS/통신사 API 연동 없음 (모킹).
- 서버 검증/중복확인 실제 호출 없음.
- 일반회원 상세 정보 입력 폼은 만들지 않음 (분기만 표시).
- 비밀번호 정책/유효성 실시간 검증은 최소화.
